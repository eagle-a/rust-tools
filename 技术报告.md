# Rust 缓存清理工具 - 技术报告

## 一、项目概述

Rust 缓存清理工具是一个用于管理和清理 Rust 开发环境缓存的桌面应用程序。

### 技术栈
- **前端**: Vue 3 + TypeScript + Vite
- **后端**: Rust + Tauri 2
- **构建工具**: npm + Cargo

### 2.2 前端架构 (Vue 3)

#### 2.2.1 状态管理

```typescript
// 核心响应式状态
const activeTab = ref<'cache' | 'mirror' | 'logs'>('cache')  // 当前Tab
const isDark = ref(true)           // 主题模式
const cacheItems = ref<CacheItem[]>([])  // 缓存列表
const logs = ref<string[]>([])      // 日志
const isScanning = ref(false)       // 扫描中状态
const isCleaning = ref(false)        // 清理中状态
```

#### 2.2.2 核心函数逻辑

**1. initTheme() - 主题初始化**
```typescript
function initTheme() {
  const now = new Date();
  const hour = now.getHours();
  // 早上8点到晚上6点（18点）为亮色模式
  // 其他时间为暗色模式
  isDark.value = hour < 8 || hour >= 18;
  document.body.setAttribute('data-theme', isDark.value ? 'dark' : 'light');
}
```

**2. scanCache() - 扫描缓存**
- 调用 Rust 后端 `scan_cache` 命令
- 异步获取缓存项列表
- 更新缓存数量和大小统计

**3. cleanCache() - 清理缓存**
- 获取选中的缓存项
- 调用 Rust 后端 `clean_cache` 命令
- 清理完成后自动重新扫描

**4. toggleItem() - 切换选中状态**
- 带边界检查，防止数组越界
- 实时更新选中大小统计

### 2.3 后端架构 (Rust)

#### 2.3.1 核心数据结构

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CacheItem {
    pub name: String,           // 缓存名称
    pub path: String,          // 完整路径
    pub relative_path: String,  // 相对路径（用于UI显示）
    pub size: u64,             // 大小（字节）
    pub item_type: String,      // 类型（注册表/Git/Target等）
    pub last_modified: Option<String>,  // 最后修改时间
    pub selected: bool,         // 是否选中
}
```

#### 2.3.2 主要命令

**1. scan_cache 命令**
- 扫描 `~/.cargo/registry` - 注册表缓存
- 扫描 `~/.cargo/git` - Git 依赖缓存
- 扫描 `~/.rustup` - Rustup 工具链
- 扫描 `./target` - 项目编译产物
- 返回 `ScanResult` 包含所有缓存项和统计信息

**2. clean_cache 命令**
- 接收要清理的路径列表
- 检查路径是否存在
- 递归计算目录大小
- 删除文件或目录
- 返回清理结果和日志

**3. check_mirror 命令**
- 检查 `~/.cargo/config.toml`
- 识别镜像源类型（清华/中科大/RSProxy/默认）
- 返回镜像配置信息

### 2.4 主题切换机制

使用 CSS 变量实现主题切换：

```css
:root {
  --bg-primary: #1e1e1e;  /* 暗色模式 */
  --bg-secondary: #232323;
}

body[data-theme="light"] {
  --bg-primary: #f5f5f5;  /* 亮色模式 */
  --bg-secondary: #ffffff;
}
```

## 三、Bug 修复记录

### 3.1 已修复的 Bug

| Bug 描述 | 修复方案 |
|---------|---------|
| clean_cache 不检查路径是否存在 | 添加路径存在性检查，不存在则跳过 |
| .cargo 目录被重复扫描 | 移除对 .cargo 根目录的扫描，只扫描子目录 |

## 五、使用说明

### 5.1 运行应用

```bash
# 开发模式
cd rust-cache-cleaner
npm run tauri dev

# 或直接运行 exe
./rust-cache-cleaner.exe
```

### 5.2 功能操作

1. **扫描缓存**: 点击"扫描缓存"按钮
2. **选择清理项**: 勾选列表项或使用全选/取消全选
3. **清理**: 点击"清理选中"按钮
4. **查看镜像**: 切换到"镜像源"标签页
5. **查看日志**: 切换到"日志"标签页

### 5.3 主题说明

- 自动模式：根据系统时间自动选择主题（8:00-18:00 为亮色）
- 手动切换：点击导航栏右侧的主题切换按钮

1. 清理缓存后会重新下载依赖，首次编译会较慢

3. 不会删除 Rust 工具链本身
